# 💇‍♀️ 理发店AI记账助手 - 产品需求文档 (PRD)

## 📋 产品概述

### 产品名称
**账掌柜** - AI智能记账助手

### 产品定位
面向理发店等个体工商户的移动端AI记账应用，通过自然语言对话完成所有业务操作，让不懂技术的小商家也能轻松管理店铺。

### 目标用户
- 理发店个体户
- 美容美发行业小商家
- 中小型服务业商户
- 年龄35-60岁，对技术不熟悉的店主

### 核心理念
**"说话就能记账，聊天就能管店"**

## 💡 核心功能

### 1. AI对话交互（核心）

#### 1.1 自然语言记账
```
用户: "刚收了一个洗剪吹38块"
AI: "好的，已记录收入38元，服务项目：洗剪吹。今日累计收入238元。"

用户: "老李烫头收了280"
AI: "已记录，老李烫头280元。需要备注会员信息吗？"

用户: "今天进了一箱洗发水花了150"
AI: "已记录支出150元，库存：洗发水+1箱。"
```

#### 1.2 智能查询
```
用户: "这个月赚了多少？"
AI: "本月（12月）总收入18,560元，支出2,340元，净利润16,220元。
    比上月增长12.5%📈 [显示图表]"

用户: "洗发水还剩多少？"
AI: "洗发水库存：5瓶，预计还能用10天。建议下周补货。"

用户: "今天忙不忙？"
AI: "今日接待顾客15人，收入680元。下午3-5点是高峰期，接待了8人。"
```

#### 1.3 智能提醒
```
AI: "老板，洗发水只剩2瓶了，上次进货是找'美发批发王'，要帮您联系补货吗？"
AI: "今天是月末，本月收入较上月减少5%，主要是烫染项目减少，要看详细分析吗？"
```

### 2. 支付渠道集成

#### 2.1 支持的支付方式
- ✅ 微信支付（扫码收款）
- ✅ 支付宝（扫码收款）
- ✅ 现金记录
- ✅ 银行卡/刷卡

#### 2.2 自动对账
- 自动同步微信/支付宝收款记录
- 智能匹配交易与服务项目
- 每日自动生成对账报告

### 3. 收入管理

#### 3.1 收入记录
- 服务项目分类（洗剪吹、烫发、染发、护理等）
- 会员/散客标记
- 员工业绩统计
- 支付方式记录

#### 3.2 数据统计
- 日/周/月/年收入报表
- 同比环比分析
- 热门项目排行
- 客流量分析
- 员工业绩排名

### 4. 库存管理

#### 4.1 商品管理
- 洗护产品库存
- 美发工具记录
- 耗材使用追踪

#### 4.2 智能补货
- 库存预警提醒
- 历史进货记录
- 供应商信息管理
- 一键生成采购清单

### 5. 会员管理

#### 5.1 会员档案
- 姓名/电话/备注
- 消费历史
- 偏好记录（发型师、服务项目等）

#### 5.2 会员营销
- 生日提醒
- 沉睡客户唤醒
- 消费积分

## 📱 界面设计

### 设计原则
1. **极简操作** - 打开即聊，无需学习
2. **大字体** - 适合中老年用户
3. **语音优先** - 支持语音输入
4. **视觉清晰** - 重要数据一目了然

### 主要界面

#### 1. 首页 - 对话界面
- 类似微信聊天的对话界面
- 顶部显示今日关键数据（收入/客流）
- 底部输入框 + 语音按钮
- 常用快捷指令（可选）

#### 2. 数据看板（可通过对话唤起）
- 收入趋势图
- 项目占比饼图
- 关键指标卡片

#### 3. 设置页面
- 店铺信息
- 支付账号绑定
- AI助手偏好设置

## 🔧 技术架构

### 前端技术栈
- **框架**: React Native + Expo
- **状态管理**: Zustand
- **UI组件**: 自定义组件 + NativeWind
- **图表**: react-native-chart-kit

### 后端技术栈
- **运行时**: Node.js
- **框架**: Express.js
- **数据库**: PostgreSQL + Redis
- **AI服务**: 通义千问/智谱AI API

### 支付集成
- 微信支付商户API
- 支付宝开放平台API

## 📊 数据模型

### 核心实体
```
Transaction（交易记录）
├── id
├── type（收入/支出）
├── amount（金额）
├── category（分类）
├── paymentMethod（支付方式）
├── customerName（顾客姓名）
├── staffName（服务员工）
├── note（备注）
├── createdAt（时间）

Inventory（库存）
├── id
├── name（商品名）
├── quantity（数量）
├── unit（单位）
├── alertThreshold（预警阈值）
├── supplier（供应商）

Customer（会员）
├── id
├── name
├── phone
├── totalSpent（累计消费）
├── visitCount（来店次数）
├── lastVisit（最近到店）
├── preferences（偏好）
```
